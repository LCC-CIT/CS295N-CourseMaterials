Iterative Development
Run your app without debugging to have changes to all code applied automatically when you refresh the page

Debugging with breakpoints
Only changes to views will be updated automatically when you refresh the page, but you will be able to set breakpoints

Publishing to Azure
  • Get an Azure account using one of these two options:
    • Sign up for a free one year account (anyone can do this, not just students). Here's the link: https://azure.microsoft.com/en-us/free/. Note that this requires a credit card for identification even though the card won't be billed.
    • Use your free one year Microsoft Imagine Azure subscription through your OnTheHub account at: https://e5.onthehub.com/WebStore. This option does not require a credit card.
  • Use the publishing wizard in Visual Studio to publish your web app and create an the Azure App Service
  • View the published app in a browser. Here's my app: https://goodbooknook.azurewebsites.net
  • View the app in your Azure Dashboard: https://portal.azure.com
    • Find the "Browse" link
    • Find the "Deployment Center" (in the menu on the left), click on "FTP", then click on "Dashboard"
    • Use the FTPS credentials to view your files

Development and Debugging Tools
  • FTP clients
    • FileZilla - A popular cross-platform FTP Client
    • Net2FTP - A web based FTP client you can use to bypass the campus firewall
    • AnyClient - Another web based FTP client
  • HTTP /  HTTPS traffic monitoring
    • Fiddler - A free web debugging proxy
    • Firefox Web Console - Logs information associated with a web page: network requests, JavaScript, CSS, security errors and warnings, etc.
